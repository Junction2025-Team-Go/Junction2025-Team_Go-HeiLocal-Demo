# hei!local 🗺️

Junction 2025 해커톤 - 위치 기반 맛집 탐색 웹 앱

## 🚀 빠른 시작

### 1. Google Maps API 키 설정

`.env` 파일에 API 키를 추가하세요:

```bash
VITE_GOOGLE_MAPS_API_KEY=여기에_구글맵_API키_입력
```

### 2. 개발 서버 실행

```bash
npm run dev
```

브라우저에서 http://localhost:5173 접속

## 📍 내 위치 확인 방법

1. **브라우저에서 위치 권한 허용**
   - Chrome: 주소창 왼쪽 자물쇠 → 위치 허용
   - Safari: 설정 → 웹사이트 → 위치 서비스

2. **내 위치가 감지되면:**
   - ✅ 지도에 녹색 마커 표시
   - ✅ 오른쪽 위 "📍 내 위치" 버튼 활성화
   - ✅ 콘솔에 "✅ 현재 위치" 출력

3. **위치 감지 실패 시:**
   - 기본 위치 (Espoo, Finland)로 설정
   - ⚠️ 콘솔에 경고 메시지

4. **"📍 내 위치" 버튼 클릭:**
   - 지도가 내 위치로 이동
   - 줌 레벨 14로 확대

## 🧪 테스트 방법

### 1. 필터 테스트
```
1. "Restaurants" 클릭 → 5개 장소만 표시 (좌하단 확인)
2. "Coffee" 클릭 → 3개 장소만 표시
3. "Favorites" 클릭 → 4개 장소만 표시 (⭐ 있는 것들)
4. 같은 버튼 다시 클릭 → 전체 10개 표시
```

### 2. 지도 상호작용
```
1. 파란색 핀 클릭 → 빨간색으로 변경 + 바운스
2. 좌측에 선택된 장소 정보 표시
3. "📍 내 위치" 버튼 클릭 → 내 위치로 이동
```

### 3. 콘솔 확인
```
F12 (개발자 도구) → Console 탭
- 필터 변경 시: "🔍 필터: ..."
- 핀 클릭 시: "📍 선택된 장소: ..."
- 위치 감지 시: "✅ 현재 위치: ..."
```

## 🐛 문제 해결

### Google Maps가 안 보여요
1. `.env` 파일 확인
2. `VITE_GOOGLE_MAPS_API_KEY` 설정 확인
3. Google Cloud Console에서 API 활성화 확인
4. 개발 서버 재시작

### 위치 감지가 안돼요
1. HTTPS 필요 (localhost는 HTTP도 가능)
2. 브라우저 위치 권한 확인
3. 기본 위치 (Espoo)로 표시되는지 확인

### 필터가 작동 안 해요
1. 콘솔에서 에러 확인
2. 디버그 정보 (좌하단) 확인
3. Mock 데이터에 해당 카테고리 있는지 확인
